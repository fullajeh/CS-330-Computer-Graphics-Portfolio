# 3D Scene Project Reflection: Computer Desk Environment

## Design Approach

I approach designing software by starting with a clear mental model of the final product, then breaking it down into smaller, manageable components. For the 3D desk scene, this meant visualizing the full setup first, including the desk, monitor, and surrounding objects, before focusing on individual models and their placement. I prioritize both functionality and user experience, even in visual projects, making sure the scene feels natural and logically structured rather than just technically correct.

## Design Skills Developed

This project helped me strengthen spatial reasoning and scene composition skills. I became more intentional about object scaling, positioning, and proportion, which are critical in making a 3D environment feel realistic. I also improved my ability to think in terms of modular design, where each object is built independently but contributes to a cohesive whole.

## Design Process

My design process followed an iterative structure: planning, prototyping, testing, and refining. I started with a basic layout of the scene, then gradually added detail and complexity. Instead of aiming for perfection upfront, I built a rough version first and improved it over time. This approach made it easier to identify what worked and what needed adjustment without getting stuck early on.

## Applying Design Tactics in the Future

The modular and iterative approach I used can be applied to future projects, especially in game development and UI design. Breaking systems into smaller components makes complex problems more manageable, while iterative refinement ensures continuous improvement. These tactics will be especially useful when building larger-scale applications or interactive environments.

---

## Development Approach

When developing programs, I focus on building a working foundation first, then expanding functionality step by step. For the 3D scene, this meant starting with basic shapes and transformations before layering in more advanced features like lighting and textures. I prioritize readability and structure in my code so that it remains maintainable as complexity increases.

## Development Strategies Used

One key strategy I used was incremental development. Instead of trying to implement everything at once, I added features in stages and tested frequently. I also relied on debugging and visual feedback to guide improvements, especially when adjusting object placement or rendering behavior.

## Role of Iteration

Iteration played a major role in this project. Almost every part of the scene went through multiple revisions, from object positioning to overall layout. Each iteration brought the scene closer to a more polished and realistic result. Without this process, small issues would have compounded into larger problems.

## Evolution of My Coding Approach

Throughout the milestones, my approach to coding became more structured and intentional. Early on, I focused mainly on getting things to work. By the end, I was more concerned with writing cleaner, more organized code and thinking ahead about scalability. I also became more comfortable troubleshooting issues and refining my logic rather than restarting from scratch.
